ADAS · HiL

ADAS Hardware-in-the-Loop Validation

Development engineer on a mini Hardware-in-the-Loop closed-loop test bench for advanced driver-assistance functions on a premium electric vehicle platform.

  • Role Development Engineer
  • Domain ADAS · HiL testing
  • Method Keyword-Driven Testing
  • Stack CANoe · TPT · CarMaker

The challenge

Validate driver-assistance functions such as adaptive cruise and hands-free driving safely and reproducibly, without putting a real vehicle on the road for every test.

The approach

A mini Hardware-in-the-Loop closed-loop bench: rest-bus simulation around the real ECUs and closed-loop co-simulation of the vehicle and its environment, with test cases written in a structured, repeatable way.

1

Design test cases

Keyword-Driven Testing with the Precondition-Action-Expected pattern

2

Build the bench

Rest-bus simulation around the real ECUs, closed-loop co-simulation

3

Execute

Both manually and through TPT-driven automation

4

Analyse

Traces, expected-versus-actual, defect reports

Methodology

Keyword-Driven Testing and the Precondition-Action-Expected pattern keep test cases structured, readable and repeatable.

Closed loop

The vehicle and environment are co-simulated, so the real ECUs react to a simulated world instead of a static stimulus.

Result

A reproducible validation setup for adaptive-cruise and hands-free functions, with test cases that run both by hand and automated, exercising the real ECUs against a simulated vehicle and environment.
