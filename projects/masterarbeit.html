Cooperative-Driving Occupancy Grid

Master's thesis: a custom Nav2 costmap-layer plugin in C++ that keeps a vehicle out of the space a cooperating vehicle is about to occupy, even when its own sensors have no line of sight.

  • Role Sole author (thesis)
  • Context ARRK Engineering, Motion Planning
  • Stack ROS2 Humble · Nav2 · C++ / Python
  • Grade M.Eng. Automated Driving & Vehicle Safety, THI

The challenge

Reason about space the ego's own sensors cannot see, and keep the vehicle out of the area a cooperating vehicle will pass through a few seconds later, using a standard navigation stack.

The idea

Turn the shared trajectory into a geometric occupancy region and feed it to Nav2 as an extra cost layer, so the standard planner avoids it with no change to its own logic.

V2V exchange

polygon_cost_layer.cpp 
// custom Nav2 costmap layer: stamps the area the
// cooperating vehicle will occupy onto the master grid
void PolygonCostLayer::updateCosts(
    nav2_costmap_2d::Costmap2D & master_grid,
    int min_i, int min_j, int max_i, int max_j)
{
  clampBoundsToGrid(master_grid, min_i, min_j, max_i, max_j);

  const auto data = prepareObstacleData();  // V2V polygons -> cells
  if (data.empty()) return;

  applyCostsToGrid(master_grid, data,
                   min_i, min_j, max_i, max_j);
}

// registered as a Nav2 plugin, loaded at runtime
PLUGINLIB_EXPORT_CLASS(
    polygon_cost_layer::PolygonCostLayer,
    nav2_costmap_2d::Layer)
Result

A working plugin that keeps the ego out of the cooperating vehicle's predicted path in straight-lane and oval scenarios, validated in simulation, with a reproducible C++/ROS2 codebase (CI/CD, gtest/pytest).
